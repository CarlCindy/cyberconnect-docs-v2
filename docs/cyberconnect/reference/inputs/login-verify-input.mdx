---


The `LoginVerifyInput` input type specifies the Typed Data message parameters along with it's signature.

```graphql
input LoginVerifyInput {
  domain: String!
  address: AddressEVM!
  chainID: ChainID!
  signature: String!
  isEIP1271: Boolean
}
```


### Fields

#### `domain` ([`String`](/cyberconnect/reference/scalars/string))

`domain` the domain of the app.


#### `address` ([`AddressEVM`](/cyberconnect/reference/scalars/address-evm))

`address` the address of the user.


#### `chainID` ([`ChainID`](/cyberconnect/reference/scalars/chain-id))

`chainID` the chain ID of the network.


#### `signature` ([`String`](/cyberconnect/reference/scalars/string))

`signature` the signature of the message.


#### `isEIP1271` ([`Boolean`](/cyberconnect/reference/scalars/boolean))

`isEIP1271` whether verify signature in the way of EIP-1271.
