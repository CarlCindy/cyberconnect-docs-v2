Тип `Profile` предоставляет информацию о профиле.

```graphql
type Profile implements Node {
  id: ID!
  profileID: Int!
  handle: String!
  owner: Wallet!
  metadata: URL
  metadataInfo: MetadataDetail
  avatar: URL
  mintingBlock: Int!
  isPrimary: Boolean!
  operators: [AddressEVM!]!
  essences(
  after: Cursor
  first: Int
  before: Cursor
  last: Int
  orderBy: EssenceOrder
): EssenceConnection!
  subscribers(
  after: Cursor
  first: Int
  before: Cursor
  last: Int
  orderBy: SubscribeOrder
): SubscribeConnection!
  subscribeNFT: ContractAddressEVM
  subscribeMw: MiddlewareInfo
  namespace: Namespace!
  subscriberCount: Int!
}
```


### Поля

#### `id` ([`ID`](/reference/scalars/id))

`id` - уникальный идентификатор профиля.


#### `profileID` ([`Int`](/reference/scalars/int))

`profileID` - идентификатор токена профиля.


#### `handle` ([`String`](/reference/scalars/string))

`handle` - дескриптор профиля.


#### `owner` ([`Wallet`](/reference/objects/wallet))

`owner` - кошелек владельца профиля.


#### `metadata` ([`URL`](/reference/scalars/url))

`metadata` - URL-адрес метаданных профиля.


#### `metadataInfo` ([`MetadataDetail`](/reference/objects/metadata-detail))

`metadataInfo` - информация о метаданных профиля.


#### `avatar` ([`URL`](/reference/scalars/url))

`avatar` - URL-адрес аватара профиля.


#### `mintingBlock` ([`Int`](/reference/scalars/int))

`mintingBlock` - номер блока, когда профиль был отчеканен.


#### `isPrimary` ([`Boolean`](/reference/scalars/boolean))

`isPrimary` - является ли профиль основным профилем владельца.


#### `operators` ([`AddressEVM`](/reference/scalars/address-evm))

`operators` - список адресов операторов в профиле.


#### `essences` ([`EssenceConnection`](/reference/objects/essence-connection))

`essences` - предоставляет постраничный список упорядоченных сущностей.
- ##### `after` ([`Cursor`](/reference/scalars/cursor))

`after` - курсор для получения результатов после.


- ##### `first` ([`Int`](/reference/scalars/int))

`first` - максимальное количество сущностей, возвращаемых послекурсора.


- ##### `before` ([`Cursor`](/reference/scalars/cursor))

`before` - курсор для получения результатов ранее.


- ##### `last` ([`Int`](/reference/scalars/int))

`last` - максимальное количество сущностей, возвращаемых перед курсором.


- ##### `orderBy` ([`EssenceOrder`](/reference/inputs/essence-order))

`orderBy` -  порядок, в котором возвращаются результаты.


#### `subscribers` ([`SubscribeConnection`](/reference/objects/subscribe-connection))

`subscribers` - предоставляет постраничный список упорядоченных подписчиков.
- ##### `after` ([`Cursor`](/reference/scalars/cursor))

`after` - курсор для получения результатов после.


- ##### `first` ([`Int`](/reference/scalars/int))

`first` - максимальное количество подписчиков, возвращаемых после курсора.


- ##### `before` ([`Cursor`](/reference/scalars/cursor))

`before` - курсор для получения результатов ранее.


- ##### `last` ([`Int`](/reference/scalars/int))

`last` - максимальное количество подписчиков, возвращаемых перед курсором.

- ##### `orderBy` ([`SubscribeOrder`](/reference/inputs/subscribe-order))

`orderBy` - порядок, в котором возвращаются результаты.


#### `subscribeNFT` ([`ContractAddressEVM`](/reference/scalars/contract-address-evm))

`subscribeNFT` - адрес контракта Subscribe NFT профиля.


#### `subscribeMw` ([`MiddlewareInfo`](/reference/objects/middleware-info))

`subscribeMw` - информация о промежуточной программе подписки профиля.


#### `namespace` ([`Namespace`](/reference/objects/namespace))

`namespace` - информация о пространстве имен профиля.


#### `subscriberCount` ([`Int`](/reference/scalars/int))

`subscriberCount` - количество подписчиков.


### Интерфейсы

#### [`Node`](/reference/interfaces/node) 

Интерфейс `Node` соответсвует спецификацим relay.dev.
